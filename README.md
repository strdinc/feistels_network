# Сеть Фейстеля на Flask

Веб-приложение демонстрирует работу сети Фейстеля с 8 раундами и правосторонним
XOR, дополненной гибридной криптосистемой на основе RSA. Интерфейс и код снабжены
подробными комментариями на русском языке. Приложение позволяет генерировать
пары RSA-ключей, шифровать и расшифровывать сообщения, а также экспортировать и
импортировать результаты в один JSON-файл.

## Основные возможности

- **Сеть Фейстеля** с фиксированным размером блока 16 бит, длиной ключа 16 бит и
  8 раундами.
- **Гибридная схема**: симметричный 16-битный ключ шифруется с помощью RSA.
- **Режим обратной связи**: каждый следующий блок перед шифрованием XOR-ится с
  предыдущим шифроблоком (аналог CBC).
- **Поддержка русского языка** благодаря пользовательскому алфавиту и
  преобразованию текста в биграммы, затем в двоичный вид.
- **Экспорт/импорт** зашифрованных сообщений и ключей в формате JSON, что
  облегчает передачу данных.
- **Удобный веб-интерфейс** на Flask с пояснениями для пользователя.

## Запуск проекта

```bash
python -m venv venv
```

## Описание модулей и функций

### `feistel.py`

- `ALPHABET` — строка с поддерживаемым алфавитом (русские и латинские буквы,
  цифры и базовые символы).
- `PADDING_SYMBOL` — символ дополнения, чтобы длина текста была чётной.
- `PreparedText` — dataclass с результатами подготовки текста: список блоков и
  исходная длина текста.
- `_validate_symbol(symbol)` — проверка, что символ входит в алфавит.
- `prepare_plaintext(plaintext)` — превращает текст в список 16-битных блоков:
  * переводит символы в верхний регистр;
  * объединяет в биграммы;
  * каждая биграмма переводится в десятичное число, затем в двоичный вид.
- `restore_plaintext(blocks, original_length)` — обратное преобразование блоков
  в текст (с учётом исходной длины для удаления дополнения).
- `left_rotate_bits(value, shift)` — циклический сдвиг двоичной строки влево.
- `round_function(right_half, round_key)` — раундовая функция F: XOR правой
  половины с частью ключа и сложение по модулю 256.
- `feistel_rounds(block, key)` — восемь раундов шифрования блока (правосторонний
  XOR: новая правая часть получается как XOR левой половины и F).
- `feistel_inverse_rounds(block, key)` — обратный процесс расшифрования блока.
- `xor_blocks(first, second)` — XOR двух 16-битных строк.
- `encrypt_blocks(blocks, key)` — шифрование списка блоков с обратной связью по
  шифртексту.
- `decrypt_blocks(blocks, key)` — обратное преобразование с учётом обратной
  связи.
- `ensure_16bit_key(key)` — проверка корректности ключа (ровно 16 бит).
- `format_blocks(blocks)` и `parse_blocks(blocks_text)` — удобные функции для
  представления списка блоков в текстовом виде и обратно.

### `rsa_utils.py`

- `RSAKeyPair` — dataclass для хранения открытого и закрытого ключей.
- `_is_prime(candidate)` — простая проверка числа на простоту.
- `_generate_prime(min_value, max_value)` — генерация случайного простого
  числа в диапазоне (используется для выбора p и q).
- `_egcd(a, b)` и `_mod_inverse(value, modulo)` — расширенный алгоритм Евклида
  и вычисление обратного элемента по модулю.
- `generate_key_pair()` — генерирует пару ключей RSA (открытый и закрытый).
- `encrypt(value, public_key)` и `decrypt(value, private_key)` — операции RSA.
- `serialize_key(key)` и `serialize_key_pair(key_pair)` — сохранение ключей в
  JSON-формат.
- `deserialize_key(json_text)` и `deserialize_key_pair(json_text)` — чтение
  ключей из JSON.

### `data_io.py`

- `create_package(cipher_blocks, encrypted_key, rsa_modulus, original_length, metadata)` —
  формирует словарь с данными для передачи.
- `package_to_bytes(package)` и `package_from_bytes(data)` — преобразование
  пакета в JSON и обратно.
- `package_to_filelike(package)` — создание объекта `BytesIO` (используется при
  скачивании файлов из Flask).

### `app.py`

- `index()` — единый маршрут, который обрабатывает все действия (генерация
  ключей, шифрование, расшифрование и загрузки файлов) через скрытое поле
  `action`.
- `_handle_generate_keys()` — генерирует ключи RSA и отображает их.
- `_handle_download_keypair()` — формирует файл `rsa_keypair.json` с обеими
  ключами.
- `_extract_public_key()` — извлекает открытый ключ из файла или текстового
  поля формы.
- `_handle_encrypt()` — подготавливает текст, шифрует его сетью Фейстеля,
  зашифровывает симметричный ключ RSA и формирует пакет.
- `_handle_download_package()` — выдаёт файл пакета с шифртекстом и
  зашифрованным ключом.
- `_handle_decrypt()` — принимает пакет и закрытый ключ, восстанавливает
  симметричный ключ и исходный текст.
- `_handle_download_plaintext()` — позволяет сохранить расшифрованный текст.

Каждая функция в коде сопровождается комментариями, описывающими назначение и
этапы работы.

## Формат обмена данными

Экспортируемый JSON-файл содержит поля:

- `cipher_blocks` — список 16-битных строк шифртекста;
- `encrypted_key` — симметричный ключ, зашифрованный RSA (десятичное значение);
- `rsa_modulus` — модуль RSA для проверки соответствия ключей;
- `original_length` — длина исходного текста до добавления символа дополнения;
- `metadata` — служебная информация (алфавит, режим шифрования, раунды и т. д.).

Файл пары ключей `rsa_keypair.json` содержит словари `public_key` и
`private_key` с полями `n`, `e` и `d`.

## Использование интерфейса

1. **Сгенерировать ключи RSA**. Полученный файл храните у получателя, а открытую
   часть передайте отправителю.
2. **Зашифровать сообщение**. Отправитель вводит текст, указывает открытый ключ
   и скачивает JSON-файл пакета.
3. **Передать файл**. Пакет содержит и шифртекст, и зашифрованный ключ, поэтому
   достаточно передать один JSON-документ.
4. **Расшифровать пакет**. Получатель загружает пакет и свой закрытый ключ,
   после чего получает исходный текст и симметричный ключ.

## Дополнительные замечания

- Алфавит поддерживает буквы русского и латинского алфавитов, цифры и основные
  символы пунктуации. Не поддерживаемые символы вызывают ошибку.
- Если сообщение содержит нечётное количество символов, добавляется пробел в
  конец (информация о длине сохраняется и убирается после расшифровки).
- Раундовая функция F использует комбинацию XOR и сложения по модулю 256, что
  делает преобразование нелинейным.
- Модуль RSA подбирается автоматически и подходит для шифрования 16-битного
  симметричного ключа. Для реальных задач потребуются большие числа.
