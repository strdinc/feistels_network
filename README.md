# Feistel Network Demonstration

Графическое приложение на Python демонстрирует работу сети Фейстеля c
дополнительными требованиями из задания и гибридной криптосистемой на базе
RSA.  Интерфейс построен на `tkinter`, поэтому запуск не требует установки
дополнительных библиотек.

## Возможности

* 16-битная сеть Фейстеля с 8 раундами и правосторонним XOR.
* Раундовые ключи формируются циклическим сдвигом исходного ключа влево на
  номер раунда.
* Текст предварительно преобразуется: символы Unicode кодируются в UTF‑8,
  объединяются в биграммы (по два байта) и представляются 16-битными блоками.
* Реализованы три режима обратной связи: без него, XOR с предыдущим открытым
  блоком и XOR с предыдущим шифроблоком.
* Встроенный демонстрационный RSA для безопасной передачи 16-битного
  симметричного ключа.
* Подробный вывод результатов: десятичные значения блоков и их двоичная
  форма.

## Установка и запуск

Требуется Python 3.11+ (подойдет любая версия с поддержкой `tkinter`).

```bash
python app.py
```

После запуска откроется окно с настройками шифрования, режимом обратной
связи, полями для открытого текста и шифртекста, а также блоком управления RSA.

## Работа с приложением

1. **Секретный ключ**
   * Ключ должен быть целым числом от 0 до 65535.
   * Кнопка «Сгенерировать» создаёт случайный 16-битный ключ через модуль
     `secrets`.

2. **Режим обратной связи**
   * «Без обратной связи» — каждый блок шифруется независимо.
   * «XOR с предыдущим открытым блоком» — начиная со второго блока перед
     входом в сеть Фейстеля выполняется XOR с предыдущим открытым блоком.
     При расшифровке операция зеркальная.
   * «XOR с предыдущим шифроблоком» — аналогичная схема, но используется
     предыдущий шифроблок. Это повышает диффузию между блоками.

3. **Шифрование / Расшифровка**
   * Введите текст и нажмите «Зашифровать». В поле шифртекста появятся
     десятичные значения блоков, разделённые пробелами. Ниже отображается их
     двоичное представление.
   * Чтобы расшифровать, вставьте значения шифртекста (через пробел или
     запятую) и нажмите «Расшифровать».

4. **Гибридная криптосистема (RSA)**
   * «Сгенерировать RSA ключи» создаёт пару ключей (модуль примерно 512 бит).
   * «Шифровать ключ» зашифровывает текущий 16-битный секретный ключ
     открытым ключом RSA и выводит результат.
   * «Расшифровать ключ» принимает числовой шифртекст ключа и восстанавливает
     исходный ключ с помощью приватной экспоненты.

## Модуль `feistel_cipher`

### `rotate_left(value, shift, bit_width=16)`
Циклический сдвиг влево на `shift` позиций для заданной разрядности.

### `round_function(right_half, round_key)`
Раундовая функция F. Входные аргументы — правое полублок и раундовый ключ.
Функция складывает несколько побитовых преобразований и возвращает 8-битное
значение для XOR с левым полублоком.

### `generate_round_keys(master_key)`
Формирует список из восьми раундовых ключей путём циклического сдвига исходного
ключа влево на номер раунда.

### `encrypt_block(block, master_key)` / `decrypt_block(block, master_key)`
Шифрование и расшифровка отдельных 16-битных блоков.

### `text_to_blocks(text)` / `blocks_to_text(blocks)`
Преобразуют Unicode-текст в 16-битные блоки и обратно. Текст кодируется в
UTF‑8, затем байты объединяются попарно (биграммы). Если длина нечётная,
добавляется нулевой байт.

### `blocks_to_binary_strings(blocks)`
Представление блоков в виде строк из 16 двоичных символов.

### `encrypt_text(text, master_key, feedback_mode)` / `decrypt_text(cipher_text, master_key, feedback_mode)`
Высокоуровневые функции. Внутри вызывается `_process_blocks`, который
учитывает выбранный режим обратной связи, передаёт блоки через сеть Фейстеля и
возвращает как числовые значения, так и строки в бинарной записи.

### `FeedbackMode`
Простейший перечислимый класс с тремя режимами обратной связи.

### `CipherResult`
Дата-класс для удобной упаковки результатов: список блоков, их двоичное
представление и строковая форма.

## Модуль `hybrid_crypto`

### `generate_secret_key()`
Создаёт случайный 16-битный ключ для сети Фейстеля.

### `generate_rsa_keypair(prime_bits=256)`
Генерирует пару RSA-ключей. Используется показатель `65537` и алгоритм
Миллера–Рабина для поиска простых чисел. Возвращает `RSAKeyPair` с публичным и
приватным ключами.

### `rsa_encrypt(value, key)` / `rsa_decrypt(value, key)`
Минимальные реализации RSA-шифрования и расшифровки для 16-битных значений.

### Вспомогательные функции
`_generate_prime`, `_is_probable_prime`, `_mod_inverse`, `_extended_gcd` —
служебные функции для поиска простых чисел и вычисления обратного элемента.

## Модуль `app`

### `FeistelApp`
Класс-наследник `tk.Tk`, создаёт все элементы интерфейса и связывает их с
логикой. Ключевые методы:

* `_generate_secret_key`, `_set_secret_key`, `_get_secret_key` — работа с
  16-битным ключом.
* `_generate_rsa_keys`, `_encrypt_secret_key`, `_decrypt_secret_key` —
  управление гибридной схемой.
* `_encrypt`, `_decrypt` — вызов функций из `feistel_cipher` и обновление
  полей интерфейса.

### `main()`
Точка входа. Создаёт экземпляр `FeistelApp` и запускает цикл событий.

## Примечания по безопасности

* Реализация RSA учебная и не оптимизирована. Для промышленного использования
  следует применять проверенные криптографические библиотеки.
* При кодировании текста используется UTF‑8. Если длина байтового
  представления нечётная, добавляется нулевой байт, который удаляется при
  расшифровке.
