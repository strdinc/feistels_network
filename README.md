# Сеть Фейстеля

## Основные возможности

- **Сеть Фейстеля** с фиксированным размером блока 16 бит, длиной ключа 16 бит и
  8 раундами.
- **Гибридная схема**: симметричный 16-битный ключ шифруется с помощью RSA.
- **Режим обратной связи**: каждый следующий блок перед шифрованием XOR-ится с
  предыдущим шифроблоком (аналог CBC).
- **Поддержка UTF-8** — можно шифровать любой текст, кодируемый в UTF-8,
  включая смешанные алфавиты и эмодзи.
- **Экспорт/импорт** зашифрованных сообщений и ключей в формате JSON, что
  облегчает передачу данных.
- **Удобный веб-интерфейс** на Flask с пояснениями для пользователя.
- **Подробные журналы** подготовки текста, работы сети Фейстеля и RSA,
  отображаемые прямо в интерфейсе.

## Запуск проекта

```bash
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
flask --app app run
```

После запуска приложение будет доступно по адресу <http://127.0.0.1:5000>.

## Описание модулей и функций

### `feistel.py`

- `ENCODING` — используемая кодировка (UTF-8).
- `PADDING_BYTE` — байт дополнения для выравнивания количества байтов.
- `PreparedText` — dataclass с 16-битными блоками и исходной длиной в байтах.
- `prepare_plaintext_with_trace(plaintext)` — подготавливает текст и возвращает
  журнал шагов; `prepare_plaintext` сохраняет прежний интерфейс.
- `restore_plaintext_with_trace(blocks, original_byte_length)` — обратное
  преобразование с журналом (есть и короткая версия `restore_plaintext`).
- `left_rotate_bits`, `round_function` — вспомогательные операции Фейстеля.
- `feistel_rounds_with_trace` и `feistel_inverse_rounds_with_trace` — шифрование
  одного блока с детальным описанием раундов (есть сокращённые версии без
  журнала).
- `encrypt_blocks_with_trace(blocks, key)` и `decrypt_blocks_with_trace(blocks, key)`
  — обработка последовательности блоков в режиме обратной связи по шифртексту
  с текстовым журналом; функции `encrypt_blocks`/`decrypt_blocks` оставлены для
  обратной совместимости.
- `ensure_16bit_key`, `format_blocks`, `parse_blocks` — служебные функции.

### `rsa_utils.py`

- `RSAKeyPair` — dataclass с объектами `RSAPublicKey` и `RSAPrivateKey`.
- `generate_key_pair(key_size)` — создание пары ключей с помощью библиотеки
  `cryptography` (по умолчанию 2048 бит).
- `serialize_public_key(key)` и `serialize_private_key(key)` — сериализация в
  PEM-формат.
- `load_public_key(pem_text)` и `load_private_key(pem_text)` — чтение ключей из
  PEM.
- `encrypt(data, public_key)` и `decrypt(token, private_key)` — RSA-OAEP над
  произвольными байтами с результатом в base64.
- `get_modulus(key)` — извлечение модуля `n` из открытого или закрытого ключа.

### `data_io.py`

- `create_package(cipher_blocks, encrypted_key, rsa_modulus, original_byte_length, metadata)` —
  формирует словарь с данными для передачи.
- `package_to_bytes(package)` и `package_from_bytes(data)` — преобразование
  пакета в JSON и обратно.
- `package_to_filelike(package)` — создание объекта `BytesIO` (используется при
  скачивании файлов из Flask).

### `app.py`

- `index()` — единый маршрут, который обрабатывает все действия (генерация
  ключей, шифрование, расшифрование и загрузки файлов) через скрытое поле
  `action`.
- `_handle_generate_keys()` — генерирует ключи RSA и отображает их.
- `_handle_download_public_key()` и `_handle_download_private_key()` —
  позволяют скачать ключи по отдельности в формате PEM.
- `_extract_public_key()` — извлекает открытый ключ из файла или текстового
  поля формы.
- `_handle_encrypt()` — подготавливает текст, шифрует его сетью Фейстеля,
  зашифровывает симметричный ключ RSA и формирует пакет вместе с журналами.
- `_handle_download_package()` — выдаёт файл пакета с шифртекстом и
  зашифрованным ключом.
- `_handle_decrypt()` — принимает пакет и закрытый ключ, восстанавливает
  симметричный ключ, текст и формирует журналы.
- `_handle_download_plaintext()` — позволяет сохранить расшифрованный текст.

Каждая функция в коде сопровождается комментариями, описывающими назначение и
этапы работы.

## Формат обмена данными

Экспортируемый JSON-файл содержит поля:

- `cipher_blocks` — список 16-битных строк шифртекста;
- `encrypted_key` — симметричный ключ, зашифрованный RSA (строка base64);
- `rsa_modulus` — модуль RSA для проверки соответствия ключей;
- `original_byte_length` — длина исходного текста в байтах до дополнения;
- `metadata` — служебная информация (алфавит, режим шифрования, раунды и т. д.).

Отдельные файлы `rsa_public_key.pem` и `rsa_private_key.pem` содержат ключи в
стандартном PEM-представлении.

## Использование интерфейса

1. **Сгенерировать ключи RSA**. Сохраните приватный ключ в безопасном месте,
   а публичный передайте отправителю.
2. **Зашифровать сообщение**. Отправитель вводит текст, указывает открытый ключ
   и скачивает JSON-файл пакета.
3. **Передать файл**. Пакет содержит и шифртекст, и зашифрованный ключ, поэтому
   достаточно передать один JSON-документ.
4. **Расшифровать пакет**. Получатель загружает пакет и свой закрытый ключ,
   после чего получает исходный текст и симметричный ключ.

## Дополнительные замечания

- Текст переводится в байты UTF-8, поэтому поддерживаются любые символы,
  представимые в этой кодировке.
- Если количество байтов нечётное, добавляется нулевой байт. Информация о
  исходной длине сохраняется, и после расшифровки дополнение удаляется.
- Раундовая функция F использует комбинацию XOR и сложения по модулю 256, что
  делает преобразование нелинейным.
- RSA использует модуль 2048 бит, шифрование выполняется по схеме OAEP с
  SHA-256. Для реальных задач рекомендуется защищать приватный ключ паролем.
